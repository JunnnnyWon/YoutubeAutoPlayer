# 🏫 스쿨 뮤직 타이머 (School Music Timer) 개발 요청서

## 1. 프로젝트 개요

-   **프로젝트명**: 스쿨 뮤직 타이머 (School Music Timer)
-   **목표**: 학교 점심시간 방송 자동화를 위한 데스크톱 애플리케이션 개발
-   **핵심 목적**: 방송부 담당자가 매일 수동으로 음악을 켜고 끄는 번거로움을 없애고, 설정된 스케줄에 따라 자동으로 YouTube 플레이리스트를 재생 및 종료하여 방송 업무를 완전 자동화한다.

---

## 2. 필수 기능 및 요구사항

### 2.1. 사용자 인터페이스 (GUI)
-   **언어/라이브러리**: Python / `Tkinter`
-   **화면 구성**:
    -   **대시보드**: 현재 시간과 다음 방송 예정 시간을 실시간으로 표시한다.
    -   **요일별 스케줄러**:
        -   월요일부터 금요일까지 각 요일별로 **시작 시간 (HH:MM)**과 **종료 시간 (HH:MM)**을 입력하고 저장할 수 있는 테이블 형태의 UI를 구현한다.
    -   **플레이리스트 관리**:
        -   YouTube 플레이리스트 URL을 여러 개 추가, 삭제, 관리할 수 있는 목록(Listbox)을 구현한다.
        -   저장된 목록 중 **'오늘 방송할 플레이리스트'**를 선택(라디오 버튼 또는 드롭다운)할 수 있어야 한다.
    -   **제어 버튼**:
        -   **"자동 방송 시작"**: 설정된 스케줄을 활성화하고 백그라운드 대기 모드로 진입하는 메인 버튼.
        -   **"즉시 종료 (비상 정지)"**: 현재 재생 중인 모든 작업을 즉시 중단하고 브라우저를 닫는 버튼.
    -   **예외 처리**:
        -   **"오늘은 실행 안 함"**: 활성화 시 당일 모든 스케줄을 무시하는 체크박스.
    -   **상태 표시줄**: "대기 중", "다음 방송: 12:30 (플레이리스트명)", "오류: 인터넷 연결 없음" 등 프로그램의 현재 상태를 명확히 표시한다.

### 2.2. 핵심 자동화 로직
-   **언어/라이브러리**: Python / `Selenium`, `Schedule`, `Threading`
-   **동작 흐름**:
    1.  **스케줄 활성화**: "자동 방송 시작" 버튼을 누르면, `threading`을 이용해 GUI가 멈추지 않도록 백그라운드에서 스케줄러(`schedule`)가 동작을 시작한다.
    2.  **시간 도달**: 오늘 요일에 맞는 '시작 시간'이 되면, 아래의 '방송 시작' 작업을 수행한다.
    3.  **방송 시작**:
        -   `Selenium`을 사용하여 **Chrome 웹 드라이버**를 실행한다.
        -   브라우저를 **전체 화면**으로 전환한다.
        -   지정된 YouTube 플레이리스트 URL로 접속한다. (URL 끝에 `&autoplay=1`을 추가하여 자동 재생을 시도한다.)
    4.  **방송 종료**: '종료 시간'이 되면, 실행된 Chrome 브라우저 인스턴스를 완전히 종료(`driver.quit()`)한다.

### 2.3. 설정 및 데이터 관리
-   **방식**: `configparser` 라이브러리를 사용해 설정 파일(`config.ini`)에 모든 데이터를 저장하고 불러온다.
-   **저장 데이터**:
    -   요일별 시작/종료 시간
    -   저장된 모든 플레이리스트 URL 목록
    -   기본으로 선택된 '오늘의 플레이리스트'
-   **동작**: 프로그램 시작 시 `config.ini` 파일이 존재하면 설정을 자동으로 불러오고, 없으면 기본값으로 새로 생성한다.

### 2.4. 안정성 및 오류 처리
-   **인터넷 연결 확인**: 스케줄러가 동작하기 직전, 인터넷 연결 상태를 확인하고 연결이 없으면 사용자에게 경고 메시지를 표시한다.
-   **로그 기록**:
    -   `logging` 라이브러리를 사용해 프로그램의 주요 동작(시작, 종료, 오류 발생)을 타임스탬프와 함께 `school_music_timer.log` 파일에 기록한다.
    -   이를 통해 방송 사고 발생 시 원인 추적을 용이하게 한다.
-   **프로세스 관리**:
    -   프로그램 창을 닫아도 프로세스가 종료되지 않고, **시스템 트레이(System Tray)** 아이콘으로 최소화되어 백그라운드에서 계속 실행되도록 구현한다.
    -   트레이 아이콘을 우클릭하면 "열기"와 "종료" 옵션을 제공한다.

## 3. 개발 환경
-   **프로그래밍 언어**: **Python 3.x**
-   **필수 라이브러리**:
    -   `tkinter` (GUI)
    -   `selenium` (웹 브라우저 자동화)
    -   `schedule` (작업 스케줄링)
    -   `threading` (백그라운드 작업)
    -   `configparser` (설정 파일 관리)
    -   `logging` (로그 기록)
-   **외부 의존성**: **Chrome 웹 브라우저** 및 버전에 맞는 **ChromeDriver**가 설치되어 있어야 함을 가정한다.
